Conceitos Básicos

📦 O que são Objetos?

Objetos são como caixas organizadas que guardam informações relacionadas. Cada informação tem um nome (chave) e um valor.

// Um objeto é como uma ficha de personagem const pessoa = { nome: "Ana", // 🧱 propriedade (texto) idade: 25, // 🧱 propriedade (número) ativa: true, // 🧱 propriedade (boolean) falar: function() { // 🧰 método (função) return "Olá, eu sou " + this.nome + "!"; } }; // Usando o objeto console.log(pessoa.nome); // "Ana" console.log(pessoa.falar()); // "Olá, eu sou Ana!"
💡 Analogia: Pense em objetos como fichas de personagens de RPG - cada ficha tem nome, idade, habilidades, etc. Tudo organizado em um só lugar!
🧱 Propriedades (Características)

Propriedades são as características do objeto. Como cor dos olhos, altura, nome - tudo que descreve o objeto.

// Criando um objeto carro com suas características const carro = { marca: "Toyota", // 🧱 texto cor: "azul", // 🧱 texto ano: 2023, // 🧱 número automatico: true // 🧱 verdadeiro/falso }; // 2 formas de acessar propriedades: console.log(carro.marca); // "Toyota" (notação ponto) console.log(carro["cor"]); // "azul" (notação colchetes) // Verificando se existe console.log(carro.modelo); // undefined (não existe)
💡 Quando usar cada notação:
• Use objeto.propriedade quando souber o nome
• Use objeto["propriedade"] quando o nome for variável
🧰 Métodos (Ações)

Métodos são as ações que o objeto pode fazer. Como andar, falar, calcular - são as "habilidades" do objeto.

// Objeto com diferentes tipos de métodos const calculadora = { // Método tradicional somar: function(a, b) { return a + b; }, // Método moderno (ES6) multiplicar(a, b) { return a * b; }, // Método que usa outras propriedades historico: [], salvarCalculo(resultado) { this.historico.push(resultado); return "Cálculo salvo!"; } }; // Usando os métodos console.log(calculadora.somar(2, 3)); // 5 console.log(calculadora.multiplicar(4, 5)); // 20 console.log(calculadora.salvarCalculo(25)); // "Cálculo salvo!"
💡 Diferença importante:
• Propriedade: guarda um valor
• Método: executa uma ação e pode retornar um valor

Nível Iniciante

➕ Adicionando Propriedades

Você pode adicionar novas propriedades a qualquer objeto depois de criá-lo. É como adicionar novos itens à sua mochila!

// Começando com um objeto simples const jogador = { nome: "Mario" }; // Adicionando propriedades depois jogador.pontos = 100; // Usando notação ponto jogador["vidas"] = 3; // Usando colchetes jogador.poder = "Super"; // Mais uma propriedade // Verificando o resultado console.log(jogador); // { nome: "Mario", pontos: 100, vidas: 3, poder: "Super" } // Também funciona com métodos! jogador.pular = function() { return this.nome + " está pulando! 🦘"; };
💡 Flexibilidade: JavaScript permite adicionar propriedades dinamicamente. Isso é muito útil para adaptar objetos conforme necessário!
🗑️ Removendo Propriedades

Use delete para remover propriedades que não precisa mais. Como tirar itens desnecessários da mochila.

// Inventário com item inútil const inventario = { espada: 1, escudo: 1, pocao: 5, lixo: "item inútil", pedra: "sem valor" }; // Removendo propriedades indesejadas delete inventario.lixo; delete inventario.pedra; console.log(inventario); // { espada: 1, escudo: 1, pocao: 5 } // Verificando se foi removido console.log(inventario.lixo); // undefined
⚠️ Cuidado: delete remove permanentemente! Certifique-se de que realmente não precisa da propriedade.
🔄 Percorrendo Propriedades (for...in)

Use for...in para percorrer todas as propriedades de um objeto. Como verificar todos os itens da sua mochila.

const pokemon = { nome: "Pikachu", tipo: "Elétrico", nivel: 25, hp: 100, ataques: ["Thunderbolt", "Quick Attack"] }; // Percorrendo todas as propriedades console.log("=== INFORMAÇÕES DO POKÉMON ==="); for (let propriedade in pokemon) { const valor = pokemon[propriedade]; console.log(`${propriedade}: ${valor}`); } // Resultado: // nome: Pikachu // tipo: Elétrico // nivel: 25 // hp: 100 // ataques: Thunderbolt,Quick Attack
💡 Uso prático: for...in é perfeito para processar dados quando você não sabe quais propriedades existem!
🔍 Verificando se Propriedade Existe

Antes de usar uma propriedade, verifique se ela existe! Evita erros e comportamentos inesperados.

const usuario = { nome: "João", email: "joao@email.com" }; // Método 1: operador 'in' console.log("nome" in usuario); // true console.log("telefone" in usuario); // false // Método 2: hasOwnProperty (mais seguro) console.log(usuario.hasOwnProperty("email")); // true console.log(usuario.hasOwnProperty("endereco")); // false // Método 3: verificação direta if (usuario.telefone) { console.log("Telefone:", usuario.telefone); } else { console.log("Telefone não cadastrado"); } // Usando com segurança const telefone = usuario.telefone || "Não informado"; console.log("Telefone:", telefone);
💡 Boa prática: Sempre verifique se propriedades opcionais existem antes de usá-las para evitar erros!

Nível Intermediário

🔑 Object.keys() - Pegando as Chaves

Object.keys() retorna um array com todas as chaves (nomes das propriedades) do objeto. Útil para saber quais propriedades existem!

const loja = { notebook: 2500, mouse: 50, teclado: 150, monitor: 800 }; // Obtendo todas as chaves (nomes dos produtos) const produtos = Object.keys(loja); console.log(produtos); // ["notebook", "mouse", "teclado", "monitor"] // Exemplos práticos console.log(`Temos ${produtos.length} produtos`); // 4 produtos // Verificando se produto existe if (produtos.includes("mouse")) { console.log("Temos mouse em estoque!"); } // Listando todos os produtos produtos.forEach(produto => { console.log(`Produto: ${produto}`); });
💡 Quando usar: Perfeito quando você precisa saber quais propriedades um objeto tem, sem se importar com os valores.
💎 Object.values() - Pegando os Valores

Object.values() retorna um array com todos os valores das propriedades. Ideal para cálculos e análises!

const notas = { matematica: 8.5, portugues: 9.0, historia: 7.5, ciencias: 8.0 }; // Obtendo todos os valores (as notas) const todasNotas = Object.values(notas); console.log(todasNotas); // [8.5, 9.0, 7.5, 8.0] // Calculando estatísticas const soma = todasNotas.reduce((total, nota) => total + nota, 0); const media = soma / todasNotas.length; const maiorNota = Math.max(...todasNotas); const menorNota = Math.min(...todasNotas); console.log(`Média: ${media.toFixed(2)}`); // 8.25 console.log(`Maior nota: ${maiorNota}`); // 9.0 console.log(`Menor nota: ${menorNota}`); // 7.5
💡 Casos de uso: Cálculos matemáticos, estatísticas, somas, médias - quando só os valores importam!
🔗 Object.entries() - Chaves E Valores

Object.entries() retorna pares [chave, valor] em formato de array. Combina o melhor dos dois mundos!

const ranking = { primeiro: "Ana", segundo: "Bruno", terceiro: "Carlos" }; // Obtendo pares [chave, valor] const posicoes = Object.entries(ranking); console.log(posicoes); // [["primeiro", "Ana"], ["segundo", "Bruno"], ["terceiro", "Carlos"]] // Usando com forEach (destructuring nos parâmetros) console.log("🏆 RANKING FINAL:"); posicoes.forEach(([posicao, nome]) => { console.log(`${posicao}: ${nome}`); }); // Convertendo de volta para objeto const novoRanking = Object.fromEntries(posicoes); console.log(novoRanking); // Objeto original recriado // Filtrando e transformando const top2 = posicoes .slice(0, 2) // Pega só os 2 primeiros .map(([pos, nome]) => `${nome} (${pos})`); console.log(top2); // ["Ana (primeiro)", "Bruno (segundo)"]
💡 Super útil para: Transformar objetos, filtrar dados, criar relatórios onde você precisa tanto da chave quanto do valor!
🎯 Destructuring - Extraindo Dados

Destructuring permite extrair propriedades de objetos de forma elegante. Como tirar itens específicos de uma caixa organizada!

const heroi = { nome: "Superman", poder: "Voo", cidade: "Metrópolis", identidade: "Clark Kent", idade: 35 }; // Extraindo propriedades específicas const { nome, poder, cidade } = heroi; console.log(nome); // "Superman" console.log(poder); // "Voo" console.log(cidade); // "Metrópolis" // Renomeando durante a extração const { identidade: nomeReal, idade: anos } = heroi; console.log(nomeReal); // "Clark Kent" console.log(anos); // 35 // Com valores padrão (caso não exista) const { fraqueza = "Kriptonita", altura = "Não informado" } = heroi; console.log(fraqueza); // "Kriptonita" (valor padrão) // Destructuring em funções function apresentarHeroi({ nome, poder, cidade }) { return `${nome} protege ${cidade} usando ${poder}!`; } console.log(apresentarHeroi(heroi));
💡 Vantagens: Código mais limpo, menos repetição, e você pega só o que precisa do objeto!

Nível Avançado

🏗️ Função Construtora - Criando Moldes

Funções construtoras são como moldes para criar múltiplos objetos similares. Como uma fábrica que produz carros com o mesmo modelo!

// Função construtora (sempre com letra maiúscula) function Carro(marca, modelo, ano) { // 'this' se refere ao novo objeto sendo criado this.marca = marca; this.modelo = modelo; this.ano = ano; this.ligado = false; // Método dentro do construtor this.acelerar = function() { if (this.ligado) { return `${this.marca} ${this.modelo} acelerando! 🚗💨`; } else { return `Ligue o carro primeiro! 🔑`; } }; this.ligar = function() { this.ligado = true; return `${this.marca} ${this.modelo} ligado! ✅`; }; } // Criando instâncias (objetos) usando 'new' const meuCarro = new Carro("Honda", "Civic", 2023); const outroCarro = new Carro("Toyota", "Corolla", 2022); console.log(meuCarro.ligar()); // Honda Civic ligado! console.log(meuCarro.acelerar()); // Honda Civic acelerando!
💡 Convenção: Funções construtoras sempre começam com letra maiúscula. Use new para criar instâncias!
🧬 Prototype - Compartilhando Métodos

Prototype permite adicionar métodos que todos os objetos criados pela função construtora vão compartilhar. Economiza memória!

// Adicionando métodos ao prototype (mais eficiente) Carro.prototype.frear = function() { return `${this.marca} ${this.modelo} freando! 🛑`; }; Carro.prototype.desligar = function() { this.ligado = false; return `${this.marca} ${this.modelo} desligado! ❌`; }; Carro.prototype.info = function() { return `🚗 ${this.marca} ${this.modelo} (${this.ano}) - ${this.ligado ? 'Ligado' : 'Desligado'}`; }; // Todos os carros agora têm esses métodos console.log(meuCarro.frear()); // Honda Civic freando! console.log(outroCarro.info()); // Toyota Corolla (2022) - Desligado // Verificando a cadeia de protótipos console.log(meuCarro.__proto__ === Carro.prototype); // true // Adicionando propriedades ao prototype Carro.prototype.combustivel = "Gasolina"; console.log(meuCarro.combustivel); // "Gasolina"
💡 Vantagem do Prototype: Métodos no prototype são compartilhados, economizando memória. Métodos no construtor são copiados!
👑 Classes ES6 - Sintaxe Moderna

Classes são a forma moderna de criar objetos em JavaScript. Mais limpa e familiar para quem vem de outras linguagens!

// Classe base class Animal { constructor(nome, tipo) { this.nome = nome; this.tipo = tipo; this.energia = 100; } falar() { return `${this.nome} faz algum som! 🔊`; } dormir() { this.energia = 100; return `${this.nome} dormiu e recuperou energia! �`; } } // Classe filha (herança) class Cachorro extends Animal { constructor(nome, raca) { super(nome, "Cachorro"); // Chama o construtor da classe pai this.raca = raca; this.felicidade = 100; } // Sobrescrevendo método da classe pai falar() { return `${this.nome} faz: Au au! 🐕`; } // Método específico da classe filha brincar() { this.energia -= 20; this.felicidade += 10; return `${this.nome} está brincando! 🎾 (Energia: ${this.energia})`; } } const rex = new Cachorro("Rex", "Labrador"); console.log(rex.falar()); // Rex faz: Au au! console.log(rex.brincar()); // Rex está brincando! console.log(rex.dormir()); // Rex dormiu e recuperou energia!
💡 Classes vs Funções Construtoras: Classes são mais legíveis e organizadas, mas por baixo dos panos funcionam igual!
🔒 Getters e Setters - Controle de Acesso

Getters e Setters controlam como propriedades são acessadas e modificadas. Como ter um porteiro para suas propriedades!

class ContaBancaria { constructor(titular) { this.titular = titular; this._saldo = 0; // _ indica propriedade "privada" this._historico = []; } // Getter: controla como a propriedade é lida get saldo() { return `R$ ${this._saldo.toFixed(2)}`; } // Setter: controla como a propriedade é modificada set saldo(valor) { if (typeof valor !== 'number') { console.log("❌ Saldo deve ser um número!"); return; } if (valor < 0) { console.log("❌ Saldo não pode ser negativo!"); return; } this._saldo = valor; this._historico.push(`Saldo alterado para ${this.saldo}`); } depositar(valor) { if (valor > 0) { this._saldo += valor; this._historico.push(`Depósito: +R$ ${valor.toFixed(2)}`); return `✅ Depósito realizado! Saldo atual: ${this.saldo}`; } return "❌ Valor deve ser positivo!"; } get historico() { return this._historico.slice(); // Retorna cópia do array } } const conta = new ContaBancaria("Maria"); console.log(conta.depositar(100)); // Depósito realizado! console.log(conta.saldo); // R$ 100.00 conta.saldo = -50; // Saldo não pode ser negativo! console.log(conta.historico); // Array com histórico
💡 Benefícios: Validação automática, formatação de dados, e controle total sobre como propriedades são usadas!
